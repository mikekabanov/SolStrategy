Appendix: Technical Details

Data Processing Pipeline

Data Loading: CSV file loaded with pandas, timestamps converted to UTC timezone. Data inspected for completeness and anomalies.

Multi-Timeframe Aggregation: 1-minute base data aggregated to 5-minute and 1-hour candles for feature engineering at different time horizons.

Feature Normalization: All features normalized to 0-1 range using min-max scaling before model training to improve convergence and model stability.

Model Training Details

Train-Test Split: 80% training (31,758 rows), 20% testing (7,940 rows) with chronological ordering maintained to prevent lookahead bias.

Hyperparameters: Gradient Boosting optimized with learning rate 0.1, max depth 5, 100 estimators, and min samples leaf 5.

Cross-Validation: 5-fold cross-validation confirmed model stability with minimal variance across folds.

Backtesting Methodology

Entry Logic: Position opened when ML signal = 1 (probability ≥ threshold) and concurrent position count < 10.

Exit Logic: Position closed when (1) price reaches target (3% gain), or (2) 120 minutes elapsed without hitting target, whichever occurs first.

Commission & Slippage: Not included in backtest; conservative adjustment of -0.5% to actual returns recommended for live trading.
